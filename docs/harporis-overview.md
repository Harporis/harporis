# Harporis: распределённый сканер секретов, который встроен в рабочий процесс

**Harporis** — система обнаружения секретов (API-ключей, токенов, приватных
ключей, паролей) в git-репозиториях. В отличие от привычных one-shot-сканеров,
это набор независимых сервисов поверх шины сообщений — и именно архитектура даёт
ему свойства, которых у «грепалок» нет: горизонтальное масштабирование,
наблюдаемость в реальном времени и операционную зрелость от клона до
compliance-отчёта.

## 1. Честная распределённая архитектура

Под капотом — четыре кооперирующихся сервиса поверх **NATS JetStream**:

| Сервис | Роль |
|---|---|
| **getter** | клонирует/читает репозиторий, нарезает содержимое на чанки, публикует в шину |
| **scanner** | потребляет чанки, прогоняет через детекторы, публикует находки |
| **writer** | материализует находки в отчёты (на диск / в синки) |
| **nats** | durable-шина, связывает конвейер и хранит статус-поток |

Выигрыш практический, а не «модно-микросервисный»: каждый сервис масштабируется
отдельно (узкое место в сканировании → больше реплик scanner'а); падение одного
не роняет конвейер (JetStream переотдаёт неподтверждённые сообщения); статусы и
находки — durable-потоки, к которым можно подключиться в любой момент, даже к
уже завершённому скану.

## 2. Широкое и умное обнаружение

Harporis ловит не один тип секрета, а целый спектр: AWS access keys, приватные
PEM-ключи, токены GitHub (OAuth / PAT classic / fine-grained), Stripe (test/live),
GCP (API-ключи и service-account), Slack (webhook/bot), JWT, OpenAI, Anthropic,
Twilio, SendGrid, Cloudflare, Discord, Telegram, npm, PyPI, Heroku, Square и др. —
плюс **детектор высокой энтропии** для секретов без явной сигнатуры. Каждая
находка несёт severity (LOW→CRITICAL), правило, файл, строку, байтовый офсет и
сам матч.

Сканировать можно не только текущее состояние, но и историю: `current_state`,
`full_history` (каждый коммит от init), `commit_range` (A..B), `branch_full`,
`branch_diff`. Секрет, закоммиченный и «удалённый» следующим коммитом, никуда не
делся из истории — harporis его найдёт.

## 3. Отчёты под любой процесс

Одна и та же находка отдаётся в формате, который нужен именно вашему этапу:

- **ndjson** — построчно, jq-friendly, для автоматизации;
- **sarif** — для GitHub Code Scanning / IDE;
- **html** — самодостаточный отчёт с сортировкой и фильтром в браузере;
- **xlsx** — триаж в таблицах для аудиторов;
- **pdf** — формальная сдача в compliance-папку;
- **parquet / sqlite** — для SIEM, DuckDB / Polars / Athena;
- **csv / md / pretty / json** — для PR-комментариев, терминала, скриптов.

Фильтрация по severity работает и на текстовых форматах (in-process), и на
бинарных (через `writer-rebuild`, не трогая канонический отчёт на диске).

## 4. Developer experience, который не раздражает

CLI (`harporis`) — это не «запустил и жди»:

- **`harporis watch`** — живой интерактивный TUI: таблица всех сканов флота,
  обновление в реальном времени, drill-in в любой скан с метриками, историей
  событий и **вкладкой Findings** (сами найденные секреты: severity·rule·
  file:line·preview), структурный фильтр `key:value`, мультиколоночная
  сортировка.
- **`harporis scan`** — submit с блокирующим ожиданием терминального состояния
  или `--no-wait` для fire-and-forget; авто-трансляция host-путей в монтирование
  getter'а.
- **`findings show / history / metrics / ps / logs / doctor / up / down`** —
  полноценная операционная обвязка, а не голый сканер.

Статусы текут через NATS, поэтому прогресс (blobs, chunks, bytes, errors,
secrets) виден по мере работы сервисов — а не постфактум.

## 5. Готовность к приватным репозиториям

Сканирование приватных remote-репозиториев с любой реалистичной
аутентификацией: HTTPS Basic, Bearer-токен, произвольный заголовок (например
`PRIVATE-TOKEN` для GitLab), PAT через `x-access-token`, SSH-ключ + known_hosts.
Секреты **никогда не попадают в argv** (передаются через env `GIT_CONFIG_*`), а в
логах редактируются. Можно задать `default_auth` по хостам в конфиге getter'а,
чтобы не таскать креды в каждой команде.

## 6. Простой деплой и открытое ядро

Образы публикуются в **GHCR**, всё поднимается одним `docker compose`. Есть
`.deb` / `.rpm`-пакеты CLI и shell-completions. Модель **open-core**:
Community-редакция бесплатна и самодостаточна (включая fleet-просмотр),
Pro/Enterprise добавляют масштабные дашборды, постоянное хранилище и
кросс-репличные виды.

## Итог

Harporis выигрывает там, где обычный сканер выдыхается: на **больших
репозиториях и историях** (распределённое сканирование), на **наблюдаемости**
(реальный прогресс, а не чёрный ящик), на **интеграции** (семь форматов отчётов
под семь процессов) и на **операционной зрелости** (auth, деплой, TUI, метрики).
Это не «найти AKIA в коде» — это инструмент безопасности, встроенный в рабочий
процесс от клона до compliance-отчёта.
